rg（全名 ripgrep）是一个超快的文本搜索工具，比 grep、ack、ag 都更高效，常用于代码库内容搜索。

下面我详细教你几种典型用法，以查找 ENABLE_WIFI 为例。

🔍 基本用法

在当前目录（及其子目录）中搜索 ENABLE_WIFI：

bash
rg ENABLE_WIFI

默认行为：

  • 会递归搜索所有文件（包括子目录）

  • 自动忽略 .gitignore 中排除的文件

  • 高亮匹配内容

  • 输出格式：文件路径:行号:匹配内容

🎯 精确控制搜索范围

1. 只搜索特定文件类型

例如只在 .cpp 和 .h 文件中查找：

bash
rg ENABLE_WIFI -g "*.cpp" -g "*.h"

或使用内置类型别名（--type / -t）：

bash
rg ENABLE_WIFI -t cpp

可查所有支持的类型：

bash
rg --type-list

2. 在特定目录下搜索

例如只查 src/ 目录：

bash
rg ENABLE_WIFI src/

3. 搜索时忽略大小写

bash
rg -i enable_wifi

4. 只显示文件名（不显示内容）

想知道哪些文件包含该词：

bash
rg -l ENABLE_WIFI

5. 只显示匹配的内容（不带行号或路径）

bash
rg -o ENABLE_WIFI

6. 搜索正则表达式

rg 默认使用正则表达式，例如：

bash
rg "ENABLE_WIFI|ENABLE_BT"

表示匹配 ENABLE_WIFI 或 ENABLE_BT。

7. 显示上下文（前后几行）

bash
rg ENABLE_WIFI -A 2 -B 2

  • -A 2 → 后两行

  • -B 2 → 前两行

  • -C 2 → 前后各两行

8. 高亮和颜色控制

默认高亮开启，可用：

bash
rg --color=always ENABLE_WIFI

若要输出纯文本（无颜色）：

bash
rg --color=never ENABLE_WIFI

9. 配合管道或重定向

bash
rg ENABLE_WIFI > result.txt

将结果保存到文件。

ChatGPT:

非常好，这个问题问得非常实用。
在使用 rg（ripgrep）全盘或根目录 / 搜索时，确实会遇到很多“系统目录”（如 /proc、/sys、/dev）导致搜索卡顿或报错。
我们可以通过 排除路径 的方式非常优雅地解决。

✅ 一、最常用方法：用 --glob 或 -g '!pattern' 排除目录

bash
sudo rg ENABLE_WIFI / -g '!/proc/*' -g '!/sys/*' -g '!/dev/*'

解释：

  • -g 表示全局匹配模式（glob pattern）

  • 前面的 ! 表示“排除”

  • 例如：-g '!/proc/*' 表示不搜索 /proc 下的内容

💡 你可以加多个 -g '!pattern' 来排除多个路径。

✅ 二、使用 --ignore-dir（ripgrep ≥13 已支持）

bash
sudo rg ENABLE_WIFI / --ignore-dir=proc --ignore-dir=sys --ignore-dir=dev

注意：

  • 这个选项对路径在搜索根目录下时生效（比如 /proc、/sys）

  • 等价于在 .ignore 文件中添加同名规则

✅ 三、使用 .ignore 或 .rgignore 文件（推荐在工程内使用）

如果你经常在某个项目里用 rg，可以在项目根目录下放一个 .rgignore 文件：

bash
# .rgignore
/proc/
/sys/
/dev/

然后直接运行：

bash
rg ENABLE_WIFI /

ripgrep 会自动读取并忽略这些目录。

✅ 四、综合安全示例（推荐版本）

适合你当前这种在系统级目录下搜索的情况 👇

bash
sudo rg ENABLE_WIFI / \
  -g '!/proc/*' \
  -g '!/sys/*' \
  -g '!/dev/*' \
  -g '!/run/*' \
  -g '!/tmp/*' \
  -g '!/var/log/*'

这能显著减少搜索时间并避免报错（如“Permission denied”）。

✅ 五、补充：忽略错误输出

如果仍有权限报错，可以静默忽略错误：

bash
sudo rg ENABLE_WIFI / -g '!/proc/*' -g '!/sys/*' 2>/dev/null

✅ 六、如果你只想搜索 /etc、/usr 等特定路径

比起排除法，更快更安全 的方式是直接指定目录：

bash
sudo rg ENABLE_WIFI /etc /usr

如果你想用 rg (ripgrep) 查找文件名，而不是文件内容，有两种方法：

1️⃣ 使用 -g 配合 --files

bash
rg --files -g 'network_manage.sh'

解释：

  • --files → 列出目录下所有文件（而不是搜索内容）

  • -g 'pattern' → 只匹配指定模式

如果你要在 特定目录 下搜索，比如 /home/user/：

bash
rg --files -g 'network_manage.sh' /home/user/

2️⃣ 使用 --glob 在整个系统中搜索（结合排除目录）

如果想搜索系统根目录 /，避免 /proc /sys /dev 等目录：

bash
sudo rg --files -g 'network_manage.sh' / \
  -g '!/proc/*' -g '!/sys/*' -g '!/dev/*' -g '!/run/*'

✅ 这样可以快速找到文件名，而不浪费时间搜索系统目录内容。

3️⃣ 补充：快速替代方案

如果只想找文件而不用 rg，传统方法是：

bash
find / -name 'network_manage.sh' 2>/dev/null

  • 2>/dev/null → 忽略权限报错

  • find 对查找文件名比 rg 更直观
