一、先理解你现在为什么装不上

你看到的：

W: GPG error: https://repo.huaweicloud.com/ros/ubuntu focal InRelease: ...
NO_PUBKEY F42ED6FBAB17C654
E: The repository 'https://repo.huaweicloud.com/ros/ubuntu focal InRelease' is not signed.

这不表示华为云仓库一定坏了，而是表示：

  1. APT 已经拿到了这个 ROS 仓库的元数据。

  2. 但它没有可用的公钥去验证签名。

  3. 因为无法验证签名，APT 按安全策略拒绝使用这个仓库。

  4. 仓库被禁用以后，apt install ros-noetic-rosbridge-server 就只能报 Unable to locate package。GitHub+1

所以你不是卡在“安装阶段”，而是卡在“仓库认证阶段”。

二、总体思路

我们分成 4 段来做：

  1. 清理旧的 ROS 源配置和残留 key

  2. 在另一台能联网的机器下载 ros.key

  3. 把 ros.key 传到当前机器并导入

  4. 配置华为云 ROS 源并安装包

三、在当前机器上先做清理

先打开终端，执行下面这些命令。

1）查看有没有挂起任务

你前面有过 curl | gpg 被挂起的情况，先清掉：

jobs

如果看到类似：

[1]+  Stopped ...

就杀掉它，比如：

kill %1

如果不是 %1，就按 jobs 里显示的编号改。

2）删除旧 ROS 源文件

先把可能混乱的 ROS 源删掉，只保留 Ubuntu 基础源：

sudo rm -f /etc/apt/sources.list.d/ros*.list

然后确认一下：

ls /etc/apt/sources.list.d

这里应该看不到 ros1.list、ros-latest.list 之类的 ROS 源文件。

3）删除旧 keyring 文件

把之前失败或过期的 keyring 也删掉，避免后面混用：

sudo rm -f /usr/share/keyrings/ros-archive-keyring.gpg

再确认：

ls -l /usr/share/keyrings/ros-archive-keyring.gpg

正常会提示文件不存在。

4）更新 Ubuntu 基础源

这一步只是确认系统本身没问题：

sudo apt clean
sudo rm -rf /var/lib/apt/lists/*
sudo apt update

如果这里只剩 Ubuntu 源的输出，没有 ROS 报错，说明清理成功了。

四、在另一台能联网的机器上下载 ros.key

因为你当前机器访问：

  • raw.githubusercontent.com

  • packages.ros.org

都失败，所以这一步要在别的能联网的电脑上做。

在另一台机器上下载 ros.key。拿到这个文件即可，文件名就叫：

ros.key

下载来源按 ROS 官方文档，通常就是 ROS 仓库 key 文件。官方安装流程要求先获取这个 key，再配置 apt 仓库。ROS Documentation+1

下载完以后，把这个文件传到你当前这台机器。放到哪里都可以，我建议放到：

/tmp/ros.key

你可以用 scp、U 盘、共享目录、宿主机挂载、VSCode Remote、SFTP 都行。

五、在当前机器上导入 ros.key

假设你已经把文件传到了：

/tmp/ros.key

先确认文件存在：

ls -l /tmp/ros.key

应该能看到文件大小，不应再是 No such file or directory。

1）先查看这个 key 文件内容是否有效

执行：

gpg --show-keys /tmp/ros.key

正常情况下，这条命令会显示这个 key 的信息，而不是报：

no valid OpenPGP data found

如果这里就报错，说明你传过来的 ros.key 文件本身不对，得重新下载。

2）把 ASCII key 转成 apt 用的 keyring

执行：

sudo gpg --dearmor -o /usr/share/keyrings/ros-archive-keyring.gpg /tmp/ros.key

然后确认生成成功：

ls -l /usr/share/keyrings/ros-archive-keyring.gpg

你应该能看到这个文件已经存在。

六、配置华为云 ROS 源

因为你的机器能访问华为云 Ubuntu 源，也能访问华为云 ROS 镜像地址，只是之前缺 key，所以这里优先用华为云 ROS 镜像。

创建源文件：

echo "deb [arch=arm64 signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] https://repo.huaweicloud.com/ros/ubuntu focal main" \
| sudo tee /etc/apt/sources.list.d/ros1.list > /dev/null

检查文件内容：

cat /etc/apt/sources.list.d/ros1.list

正常应输出：

deb [arch=arm64 signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] https://repo.huaweicloud.com/ros/ubuntu focal main

这里有几个点不能写错：

  • arch=arm64

  • signed-by=/usr/share/keyrings/ros-archive-keyring.gpg

  • focal

  • main

七、更新 APT，并检查 ROS 包是否已经可见

执行：

sudo apt clean
sudo rm -rf /var/lib/apt/lists/*
sudo apt update

这一步的目标是：

  • 不要再出现

    • NO_PUBKEY F42ED6FBAB17C654

    • EXPKEYSIG F42ED6FBAB17C654

    • The repository ... is not signed

  • 并且不要再出现 ROS 源 Connection failed

如果这一步成功，再检查包是否已经能被 APT 看到：

apt-cache policy ros-noetic-rosbridge-server
apt-cache policy ros-noetic-web-video-server

正常情况下，你会看到类似：

Candidate: 0.x.x-...
Version table:
...

只要出现 Candidate:，就说明 ROS 仓库已经恢复可用。

八、安装你要的包

现在再安装：

sudo apt install -y ros-noetic-rosbridge-server ros-noetic-web-video-server

如果依赖都正常，它就会开始下载和安装，不会再报：

Unable to locate package

九、安装后做最基本验证

安装完成后，先确认包确实装上了：

dpkg -l | grep ros-noetic-rosbridge-server
dpkg -l | grep ros-noetic-web-video-server

然后确认 ROS 环境：

source /opt/ros/noetic/setup.bash
echo $ROS_DISTRO

正常会输出：

noetic

还可以试一下命令是否存在：

roscore

另开一个终端：

source /opt/ros/noetic/setup.bash
roslaunch rosbridge_server rosbridge_websocket.launch

如果 rosbridge_server 能起起来，说明安装成功。

web_video_server 的常见启动方式也可以试：

source /opt/ros/noetic/setup.bash
rosrun web_video_server web_video_server

十、建议把 ROS 环境写进 shell 配置

如果你每次都要手动 source，比较麻烦。可以加到 ~/.bashrc 或 ~/.zshrc。

如果你现在用的是 zsh：

echo "source /opt/ros/noetic/setup.zsh" >> ~/.zshrc
source ~/.zshrc

如果你用 bash：

echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
source ~/.bashrc

十一、如果中途某一步失败，分别怎么看

情况 A：gpg --show-keys /tmp/ros.key 失败

说明 /tmp/ros.key 文件不对，不是有效的 PGP key。
处理办法：重新在另一台机器下载并传过来。

情况 B：apt update 仍然报 NO_PUBKEY

说明：

  • key 没正确导入

  • 或 ros1.list 里的 signed-by= 路径写错

  • 或 /usr/share/keyrings/ros-archive-keyring.gpg 不是由正确的 ros.key 生成的

这时重点检查：

ls -l /usr/share/keyrings/ros-archive-keyring.gpg
cat /etc/apt/sources.list.d/ros1.list

情况 C：apt update 改成报连接失败

说明华为云 ROS 镜像当时网络不可达，那就不是 key 问题，而是镜像网络问题。

情况 D：apt-cache policy 没有 Candidate

说明 APT 仍然没有成功读取 ROS 仓库索引，要回看 apt update 的 ROS 那一段报错。

十二、给你一套最短执行版

下面这套可以直接保存照着跑。前提是你已经把 ros.key 传到了 /tmp/ros.key。

# 1. 清理旧配置
sudo rm -f /etc/apt/sources.list.d/ros*.list
sudo rm -f /usr/share/keyrings/ros-archive-keyring.gpg

# 2. 检查 ros.key 是否存在且有效
ls -l /tmp/ros.key
gpg --show-keys /tmp/ros.key

# 3. 导入 key
sudo gpg --dearmor -o /usr/share/keyrings/ros-archive-keyring.gpg /tmp/ros.key
ls -l /usr/share/keyrings/ros-archive-keyring.gpg

# 4. 配置华为云 ROS 源
echo "deb [arch=arm64 signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] https://repo.huaweicloud.com/ros/ubuntu focal main" \
| sudo tee /etc/apt/sources.list.d/ros1.list > /dev/null

cat /etc/apt/sources.list.d/ros1.list

# 5. 更新索引
sudo apt clean
sudo rm -rf /var/lib/apt/lists/*
sudo apt update

# 6. 检查包是否可见
apt-cache policy ros-noetic-rosbridge-server
apt-cache policy ros-noetic-web-video-server

# 7. 安装
sudo apt install -y ros-noetic-rosbridge-server ros-noetic-web-video-server

# 8. 验证
source /opt/ros/noetic/setup.bash
echo $ROS_DISTRO
dpkg -l | grep ros-noetic-rosbridge-server
dpkg -l | grep ros-noetic-web-video-server
