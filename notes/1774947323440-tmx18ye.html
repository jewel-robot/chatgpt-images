四. cliproxyapi，管理gpt，配置出口api

Resin 负责注册ip，freemail 负责邮箱验证码，gpt-register 负责自动注册和拿 token，CLIProxyAPI 负责存账号池并对外提供 /v1 API。

VPS 上搭建 Resin + 代理 + CLIProxyAPI + gpt-register 的基本流程如下：

  1. 部署 Resin，给注册/OAuth 链路提供ip
  2. 部署 CLIProxyAPI,管理账号池，并提供 OpenAI 兼容的 /v1 接口
  3. 部署 gpt-register,自动注册账号、收验证码、跑 OAuth、产出 token
  4. 把 cfmail 改成适配 freemail,让 gpt-register 能用自己的临时邮箱接口收验证码
  5. 让 gpt-register 能访问 Resin
  6. 让 gpt-register 能访问 CLIProxyAPI
  7. 最后通过 CLIProxyAPI 的 /v1 API 使用账号池

第一步：先部署 Resin

后面 register 在跑注册/OAuth 时加上订阅会依赖它的多出口ip，不直接用vps的单出口，要不可能访问不到chatgpt或者不稳定。

我这里直接用的平时买的机场订阅，加进来用的美国ip，大概30多个节点，没有用家宽。

第二步：部署 CLIProxyAPI

1）管理账号池

存放和管理 auth-files，也就是账号 token 文件。

2）提供 /v1 API

把账号池包装成 OpenAI 风格接口

第三步：部署 gpt-register

  1. 创建临时邮箱
  2. 注册账号
  3. 接验证码
  4. 跑 OAuth
  5. 生成 token
  6. 上传到 CLIProxyAPI

第四步：把 cfmail 适配成 freemail

里面给了好几种其他的邮箱，但我实在有点手残，看站里的小白教程都没有耐心

所以直接让AI改了一下源码，把原来的 CfmailProvider 改成了一个 freemail 适配层,改为调用 freemail 的 /api/domains、/api/generate、/api/emails、/api/email/{id}，把返回结果映射成系统原本使用的 Mailbox 和验证码轮询流程，让它支持 freemail 这类流程：

  1. 获取可用域名列表
  2. 按域名生成邮箱
  3. 拉取邮箱里的邮件列表
  4. 根据邮件 ID 拉详情
  5. 从正文里提取验证码

第五步：把 Resin 接进 gpt-register

要让 gpt-register backend 在运行时能访问 Resin。

  1. 让 backend 和 Resin 在同一台 VPS 上可互通
  2. 如果是 Docker 部署，就让 backend 所在容器能访问 Resin 容器

第六步：把 CLIProxyAPI 接进 gpt-register

让 gpt-register 和账号池真正连起来。

在 gpt-register 配置里填上 CLIProxyAPI 的管理信息,这样 gpt-register 就能做：

1）维护现有池子
• 列出已有账号
• 清理失效账号
• 禁用/启用账号

2）上传新账号
• 注册成功后，把 token 生成 auth-file
• 上传到 CLIProxyAPI
• 让 /v1 API 立刻可用

第七步：怎么使用 /v1 API

当前面都打通后，CLIProxyAPI 就能像普通 OpenAI 接口一样使用了。

把它当成一个 OpenAI 兼容服务即可：

• 基础地址：

http://127.0.0.1:(端口)/v1

也可以反代出来给其他客户端用，key在CLIProxyAPI中可以看，至此结束。
