你们都是怎么做量化的？

告别空想，即刻开始：我的A股量化入门实践一周纪实

大家好，我是一个喜欢写代码、也对投资充满好奇的普通程序员。最近终于没再只停留在“想学量化”的阶段，而是用了一个周末加几个晚上，真正从零搭建了一个能跑通的A股量化回测系统。

我把它命名为 financial，不是什么高深模型，也没想秒杀市场，纯粹是一个帮你我这样的小白快速上手的入门工具。代码全部开源在GitHub上：https://github.com/CJ-LI-CHINA/financial

这篇文章，就想和大家聊聊我这一周的实践心得，也顺便分享这个项目怎么帮你跨出量化的第一步。

一、为什么说“开始比完美更重要”？

我以前总陷在一个循环里：想等学会高级统计学、等看懂所有技术指标、等一个“完美时机”再开始做量化。结果呢？一年过去了，我还在收藏夹里吃灰。

直到上周末我顿悟了：量化真正迷人的不是结果多牛，而是整个过程——数据清洗、策略设计、回测验证、迭代优化——是一个可以不断学习的闭环。哪怕最初策略简单到只是两条均线交叉，真正跑通全程带来的成就感，远胜于一年空想。

所以，别怕起步简陋。真正的高手，也都是从简单的print("Hello, Quant")开始的。

二、我的项目：finacidal · 极简A股量化入门框架

这不是一个“黑箱神器”，而是一个高度透明、代码可拆、便于修改的教学型项目。主要包含这几个模块：

  • 数据模块：基于AKShare公开数据源（合规免费），实现A股历史行情数据的自动下载与存储（支持CSV/PostgreSQL）。
  • 策略模块：内置了一个经典的双均线交叉策略（金叉买入死叉卖出）作为示例，代码完全开放，你完全可以替换成自己的逻辑。
  • 回测引擎：可计算收益曲线、夏普比率、最大回撤等基础指标，并支持可视化输出（使用Matplotlib绘制K线及买卖点）。
  • 详细注释：几乎每一行代码都写了注释，尤其适合Python和量化刚入门的朋友边运行边学习。

Git仓库里主要结构如下：

finacidal/
├── main.py              # 主程序入口：数据更新、回测、绘图
├── settings.py          # 数据库和策略参数配置
├── requirements.txt     # 依赖包列表
├── data/                # 本地数据存储目录
└── utils/               # 工具函数：指标计算、绘图等

三、5分钟快速上手：附详细操作指南

如果你也想要亲自运行一遍，感受一下“自己的第一个量化策略”，可以按下面步骤来（假设你已有Python基础环境）：

  1. 克隆项目：
    git clone https://github.com/CJ-LI-CHINA/finacidal.git
  2. cd finacidal
  3. 安装依赖：
    pip install -r requirements.txt
  4. 安装并配置PostgreSQL（如果不用数据库，也可修改代码存为CSV，这个后面再加）
    修改 settings.py 中的数据库连接字符串。
  5. 运行数据更新：
    python main.py --update
  6. 执行回测策略：
    python main.py --backtest

如果顺利，你就会看到第一张收益曲线图——可能绿油油，也可能红彤彤，但这都不重要，重要的是：你完成了从0到1的闭环。

遇到问题欢迎在评论区提出，我会尽力帮忙 debug～

四、一些真心话：量化不是魔术，是持续迭代

我第一版回测出来的曲线也不太好看——但这就是真实的起步阶段。通过回测我发现：

  • 过度拟合过去数据很容易“看上去很美”；
  • 交易费用和滑点对小资金策略影响很大；
  • 没有普适的策略，只有不断适配市场的模型。

而这些认知，只有真正动手做的人才会有。

五、邀请你一起前行：开源、互动、共同成长

我把自己这个项目定义为“开放实验室”，而不是什么权威产品。因此非常希望：

  • 你Fork / Star项目，自己尝试修改策略参数甚至重写策略；
  • 在GitHub Issue区提出建议或问题，我们可以一起讨论；
  • 如果你也写量化小笔记，欢迎@我或发Pull Request，期待更多人贡献代码和思路。

最后想说的是：真正阻碍我们的从来不是技术多难，而是开始的勇气。不管你是程序员还是投资爱好者，只要你想系统性地理解市场、练习纪律性投资，都可以从这样一个小项目起步。

一起上路吧，量化路上有个伴儿，总能走得更远。
