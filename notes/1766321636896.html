下面按你使用的 Codex 形态分别说怎么更新（最常见的是 Codex CLI）：

1) 更新 Codex CLI（终端里的 codex 命令）

先看当前版本：

bash
codex --version

A. 你是用 npm 装的（最常见）

bash
npm i -g @openai/codex@latest

这是官方给的升级方式。OpenAI Developers

B. 你是用 Homebrew 装的（macOS）

如果是 cask 安装（新方式），更新通常这样：

bash
brew update
brew upgrade --cask codex

如果你之前装的是旧的 formula（比如 v0.46.0 或更早），brew upgrade codex 可能升不上去，需要先卸载旧 formula，再装 cask，之后再用 --cask 升级。GitHub+1

C. 你是直接下的 GitHub Release 二进制

去最新 Release 下载对应平台的压缩包，解压后把旧的 codex 可执行文件替换掉，并确保它在你的 PATH 里。GitHub+1

升级后但版本没变？（常见排查）

看看你实际跑的是哪个 codex：

  • macOS / Linux：

    bash
    which codex

  • Windows：

    bat
    where codex

如果路径不是你刚升级的那个位置，通常是“装了多个 codex / PATH 优先级”导致的。

另外有时 shell 会缓存命令路径，macOS/Linux 可试：

bash
hash -r

2) 更新 IDE 里的 Codex 插件（VS Code / Cursor / Windsurf）

Codex IDE 扩展 一般会自动更新；也可以在 IDE 的扩展详情页手动“检查更新”。OpenAI Developers

3) Codex Web（chatgpt.com/codex 里的云端 Codex）

这种不需要你手动更新，刷新页面/更新 ChatGPT 客户端即可（能力更新由服务端发布）。GitHub+1

如果你告诉我你是在 终端还是 VS Code/Cursor 里用 Codex、以及你用的是 npm 还是 brew，我可以给你一条最短的“照抄命令”路径。
