简单说：Flow2API = 一个本地的 OpenAI 兼容服务，Chatbox 支持“自定义 OpenAI 兼容接口”，你只要把 Chatbox 指到 http://localhost:8000 就能在聊天里直接用它。

下面一步一步来 👇

一、先在本机跑起来 Flow2API

根据它的 README，Flow2API 提供的是标准的
http://localhost:8000/v1/chat/completions OpenAI 兼容接口。GitHub

方案 1：Docker（官方推荐）

bash
# 克隆项目
git clone https://github.com/TheSmallHanCat/flow2api.git
cd flow2api   # README 写的是 sora2api，多半是笔误

# 启动服务
docker-compose up -d

# 看日志（可选）
docker-compose logs -f

方案 2：本地 Python 跑

bash
git clone https://github.com/TheSmallHanCat/flow2api.git
cd flow2api

python -m venv venv
# Windows
venv\Scripts\activate
# Linux / macOS
source venv/bin/activate

pip install -r requirements.txt
python main.py

服务启动后，用浏览器打开：http://localhost:8000，默认账号密码是：

  • 用户名：admin

  • 密码：admin GitHub

第一次登录记得改密码。

二、确认一下“API Key”和“Base URL”

从 README 的 curl 示例可以看出：

bash
curl -X POST "http://localhost:8000/v1/chat/completions" \
  -H "Authorization: Bearer han1234" \
  ...

  • Base URL / API Host：http://localhost:8000

  • 完整请求路径：/v1/chat/completions

  • Authorization 头里的 token：例如 han1234，这就是 Chatbox 里要填的 API Key。GitHub

在 Flow2API 的 Web 管理后台里，你可以按项目说明添加/管理自己的 Token，之后在 Chatbox 里就用你配置的这个 Token 当 API Key（比如 han1234）。

另外，支持的模型名在 README 已经列好了，例如：GitHub

  • 图片：

    • gemini-2.5-flash-image-landscape

    • gemini-2.5-flash-image-portrait

    • gemini-3.0-pro-image-landscape

    • gemini-3.0-pro-image-portrait

    • imagen-4.0-generate-preview-landscape

    • imagen-4.0-generate-preview-portrait

  • 视频：

    • veo_3_1_t2v_fast_landscape / veo_3_1_t2v_fast_portrait 等等

这些原样填到 Chatbox 的“模型名称(Model ID)” 就行。

三、在 Chatbox 里添加自定义 Provider

前提：这一步需要 桌面版 Chatbox（Windows/Mac/Linux）。
Web / iOS / Android 因为不能直接连 localhost，无法用本机的 Flow2API。官方文档明确说本地模型（localhost 服务）只在桌面可用。DeepWiki

根据 Chatbox 官方“模型配置”文档，自定义提供方的流程是：docs.chatboxai.app+1

  1. 打开设置页面

    • 打开 Chatbox

    • 左侧栏底部点击 ⚙️ 设置

  2. 进入“模型提供方”

    • 在设置侧边栏里选择 “模型提供方”（或类似文案）

  3. 添加一个自定义提供方

    • 滚到列表底部，点击 “添加”（有些版本按钮上会写“添加 OpenAI 自定义”之类）

  4. 填基础信息
    在右侧配置面板里，按下面这样填（关键）：

    • 名称 (Name)：随便写，比如 Flow2API 本地

    • API 模式 (API Mode)：选择
      👉 “OpenAI API 兼容” / “OpenAI API Compatible”

    • API 密钥 (API Key)：填你在 Flow2API 里使用的 token，比如 han1234

    • API 主机 (API Host)：

      • 填：http://localhost:8000

    • API 路径 (API Path)：

      • 一般可以留空使用默认的 /v1/chat/completions，和 Flow2API 完全匹配。docs.chatboxai.app+1

    • 改善网络兼容性 / Use Proxy：

      • 如果你在公司网络等环境里经常连失败，可以把这个开关打开（等价于通过 Chatbox 的代理层转发）。Toutiao+1

  5. 添加模型

    • 在“模型 (Models)”区域点击 “+ 新建”/“Add Model”：

      • 模型 ID 填：比如 veo_3_1_t2v_fast_landscape 或 gemini-2.5-flash-image-landscape 这些 Flow2API 支持的模型名。

      • 能力勾选：

        • 图像模型就勾选“图片生成 / vision”

        • 视频模型可以先只勾“文本对话”，能正常跑起来再慢慢试其他能力

  6. 保存并测试

    • Chatbox 一般会自动保存配置

    • 点击右上或模型旁边的 “检查/Check” 按钮：

      • 如果配置正确，会提示连接成功

      • 报错的话，先检查：

        • Flow2API 是否在跑（本机能否打开 http://localhost:8000）

        • API Host 是否写错（http://、端口、有没有多空格）

        • API Key 是否和 Flow2API 后台保持一致

四、在聊天窗口里使用 Flow2API

  1. 回到 Chatbox 主界面，点击 “+ 新对话”。

  2. 在模型选择下拉框里，应该能看到你刚配置的提供方（比如 Flow2API 本地），展开后能看到你添加的模型 ID。

  3. 选择比如 veo_3_1_t2v_fast_landscape，照正常聊天那样发消息就可以了。

    • Flow2API 默认用的是流式输出（"stream": true），Chatbox 本身也支持流式显示，直接兼容。GitHub+1

五、常见坑 & 小提示

  1. 手机 / 网页版用不了 localhost

    • 想在手机上用 Flow2API，就得把 Flow2API 部署到一台有公网 / 局域网 IP 的服务器上，然后：

      • API Host 改成比如 https://your-domain.com

      • 记得配好 https 和防火墙端口。

  2. API Host 里的 /v1 要不要写？

    • Chatbox 默认 API 路径是 /v1/chat/completions，所以：

      • 推荐：API Host = http://localhost:8000

      • 如果你把 Host 填 http://localhost:8000/v1，那路径就会变成 /v1/v1/chat/completions，请求会 404。

  3. 模型名称必须完全一致

    • Chatbox 不会帮你“映射别名”，所以 一定要用 Flow2API README 里的原始 model name。GitHub+1
