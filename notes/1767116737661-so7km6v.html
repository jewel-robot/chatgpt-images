明白，你说的是最优化（optimization）里的“非精确搜索（inexact search）”：每一步不把子问题（比如线搜索、信赖域子问题、牛顿方程）解到最优，只要满足某些“足够好”的条件，就能保证整体收敛（甚至超线性）。

下面把常见类型按“你在近似什么”来讲。

1) 非精确线搜索（Inexact Line Search）

目标：在方向 pkp_kpk​ 上找步长 αk\alpha_kαk​，不必精确最小化 ϕ(α)=f(xk+αpk)\phi(\alpha)=f(x_k+\alpha p_k)ϕ(α)=f(xk​+αpk​)，只要满足下降与曲率条件。

1.1 Armijo（充分下降）/ Backtracking

  • Armijo 条件（充分下降）：

f(xk+αpk)≤f(xk)+c1α∇f(xk)⊤pk,0<c1<1f(x_k+\alpha p_k)\le f(x_k)+c_1\alpha \nabla f(x_k)^\top p_k,\quad 0<c_1<1f(xk​+αpk​)≤f(xk​)+c1​α∇f(xk​)⊤pk​,0<c1​<1

  • 实现：从 α=1\alpha=1α=1 开始，不满足就乘 ρ∈(0,1)\rho\in(0,1)ρ∈(0,1) 回溯（backtracking）。

  • 用途：最常见、简单稳健；配梯度下降、拟牛顿、牛顿都能用。

1.2 Wolfe / Strong Wolfe

在 Armijo 基础上再加曲率条件，避免步长太小：

  • Wolfe：

∇f(xk+αpk)⊤pk≥c2∇f(xk)⊤pk\nabla f(x_k+\alpha p_k)^\top p_k \ge c_2 \nabla f(x_k)^\top p_k∇f(xk​+αpk​)⊤pk​≥c2​∇f(xk​)⊤pk​

  • Strong Wolfe（更强，更常用）：

∣∇f(xk+αpk)⊤pk∣≤c2∣∇f(xk)⊤pk∣|\nabla f(x_k+\alpha p_k)^\top p_k| \le c_2 |\nabla f(x_k)^\top p_k|∣∇f(xk​+αpk​)⊤pk​∣≤c2​∣∇f(xk​)⊤pk​∣

其中 0<c1<c2<10<c_1<c_2<10<c1​<c2​<1。

直觉：Armijo 保证“确实降了”，Wolfe 保证“没降得过头导致步子太小/方向还没走够”。

2) 非精确信赖域（Inexact Trust-Region）

目标：在半径 Δk\Delta_kΔk​ 内最小化二次模型

mk(p)=f(xk)+gk⊤p+12p⊤Bkp,∥p∥≤Δkm_k(p)=f(x_k)+g_k^\top p+\tfrac12 p^\top B_k p,\quad \|p\|\le \Delta_kmk​(p)=f(xk​)+gk⊤​p+21​p⊤Bk​p,∥p∥≤Δk​

这里的“非精确”是：不精确解子问题，只要得到一个“足够下降”的 pkp_kpk​。

2.1 Cauchy step（最陡下降截断步）

沿 −gk-g_k−gk​ 方向在信赖域边界或最优点截断，保证一个下界级别的模型下降。

  • 优点：便宜、保证下降（理论里常做“最低要求”）。

2.2 Dogleg（常用于 Gauss-Newton / Levenberg–Marquardt）

  • 在“最陡下降步”和“牛顿步”之间走折线（dogleg）。

  • 适合：非线性最小二乘，速度快、实现经典。

2.3 截断共轭梯度（Truncated CG / Steihaug-Toint）

  • 用 CG 解 Bkp=−gkB_k p = -g_kBk​p=−gk​，但遇到：

    • 超出信赖域边界，或

    • 发现负曲率（p⊤Bkp<0p^\top B_k p<0p⊤Bk​p<0）
      就提前停并返回可行步。

  • 适合：大规模问题（矩阵不显式、只要 Hessian-vector product）。

3) 非精确牛顿（Inexact Newton）

牛顿步理论上解线性方程：

∇2f(xk) pk=−∇f(xk)\nabla^2 f(x_k)\, p_k = -\nabla f(x_k)∇2f(xk​)pk​=−∇f(xk​)

但大规模下不会精确求解，而是用迭代法（CG、GMRES 等）解到一定残差即可。

3.1 残差（forcing term）条件

典型要求：

∥∇2f(xk)pk+∇f(xk)∥≤ηk∥∇f(xk)∥\|\nabla^2 f(x_k)p_k + \nabla f(x_k)\| \le \eta_k \|\nabla f(x_k)\|∥∇2f(xk​)pk​+∇f(xk​)∥≤ηk​∥∇f(xk​)∥

  • ηk\eta_kηk​ 是“逼近精度”，也叫 forcing term。

  • 若 ηk\eta_kηk​ 逐渐变小（例如 ηk=min⁡(0.5,∥∇f(xk)∥)\eta_k = \min(0.5,\sqrt{\|\nabla f(x_k)\|})ηk​=min(0.5,∥∇f(xk​)∥​) 这类规则），可得到超线性甚至接近二次收敛（在常见条件下）。

3.2 实现层面

  • 用 CG（对称正定 Hessian）或 MINRES/GMRES（更一般情况）。

  • 每次迭代只需要 Hessian-vector product，避免显式 Hessian。

这类是“方向/梯度本身就不精确”，常见在机器学习：

但 prox 子问题不精确求解，只要满足某种误差界（常见于大规模稀疏/复合优化）。

5) 什么时候用哪种（工程直觉）

  • 你有好方向（牛顿/拟牛顿）但步长难选 → Armijo/Wolfe 线搜索。

  • 你担心模型不靠谱（非凸、二阶不稳） → 信赖域 + 截断 CG（很稳）。

  • 你想要二阶速度但问题很大 → 非精确牛顿（残差准则）+ 迭代线性求解器。

  • 数据大/噪声大 → SGD/变体（本质就是非精确梯度）。
