详细讲一下优化算法中的非精确搜索算法

AI

在数值最优化中，非精确线搜索（Inexact Line Search） 是现代算法（如梯度下降、牛顿法、拟合牛顿法）的核心组件。

与“精确线搜索”（寻找使目标函数达到绝对最小值的步长 α）不同，非精确搜索只要求步长能产生足够的函数值下降且步长不能太短。这样做是为了在计算成本和收敛速度之间取得平衡。

1. 为什么要用非精确搜索？

  • 计算成本： 精确搜索需要多次求解 f(xk​+αdk​) 的导数为零的点，这通常需要大量的函数和导数计算。

  • 收益递减： 在迭代初期，没必要为了寻找精确的最优点而浪费大量算力，只要方向正确、步长合适，算法就能稳步逼近极值点。

2. 核心准则（判别标准）

为了保证算法的全局收敛性，非精确搜索必须满足两个基本条件：

  1. 下降足够（Armijo 条件）：步长不能太大。

  2. 步长不能太小（Wolfe 或 Goldstein 条件）：防止步长过小导致迭代停滞。

A. Armijo 准则（足够下降准则）

这是最基础的条件，要求函数的减少量至少与步长和梯度的乘积成正比：

f(xk​+αdk​)≤f(xk​)+c1​α∇fkT​dk​

其中 c1​∈(0,1)，通常取一个很小的值（如 10−4）。

B. Wolfe 准则

Armijo 条件只能防止步长过大，但无法防止步长过小。Wolfe 准则在 Armijo 的基础上增加了曲率条件：

  1. Armijo 条件：f(xk​+αdk​)≤f(xk​)+c1​α∇fkT​dk​

  2. 曲率条件：∇f(xk​+αdk​)Tdk​≥c2​∇fkT​dk​

其中 c1​<c2​<1。这要求在新的点处的导数（投影到搜索方向上）要比原点处的导数“更平缓”或变为正值。

强 Wolfe 准则（Strong Wolfe Conditions）： 为了更好的稳定性，通常使用绝对值形式： ∣∇f(xk​+αdk​)Tdk​∣≤c2​∣∇fkT​dk​∣

C. Goldstein 准则

Goldstein 准则通过将函数值限制在一个“夹缝”中来控制步长：

(1−c)α∇fkT​dk​≤f(xk​+αdk​)−f(xk​)≤cα∇fkT​dk​

其中 c∈(0,0.5)。它直接限制了函数的下降量既要有上限也要有下限。

3. 常用算法实现：回溯线搜索 (Backtracking)

回溯线搜索是工程中最常用的方法，因为它实现简单且非常有效。它只检测 Armijo 条件。

算法步骤：

  1. 给定初始步长 α=1（牛顿法默认）或更大的值。

  2. 设定收缩因子 ρ∈(0,1)（常取 0.5）和常数 c1​。

  3. 循环： 检查 f(xk​+αdk​)>f(xk​)+c1​α∇fkT​dk​ 是否成立。

  4. 如果成立（说明下降不够），令 α=ρα，重复步骤 3。

  5. 一旦满足条件，输出当前 α。

4. 各种准则对比

准则名称优点缺点适用场景
Armijo (Backtracking)极其简单，计算量最小可能导致步长过小简单的梯度下降法
Wolfe保证全局收敛，有利于拟牛顿法 (BFGS)需要计算导数，实现较复杂拟牛顿法、共轭梯度法
Goldstein逻辑清晰，不需要计算新点导数可能避开了最优步长区域理论分析、不便求导的场景

5. 总结与应用建议

在实际开发中：

  • 如果你在使用 梯度下降法，通常配合 回溯线搜索 (Backtracking) 即可。

  • 如果你在使用 拟牛顿法 (L-BFGS)，则必须使用 强 Wolfe 准则，因为 BFGS 的正定性更新依赖于 Wolfe 条件的满足。

理解非精确线搜索的公式，最核心的技巧是**“把高维问题降成一维问题”**来看。

当你站在山坡上的 xk​ 点，决定往 dk​ 方向走 α 步时，原本复杂的函数 f(x) 就变成了一个关于步长 α 的一维函数：ϕ(α)=f(xk​+αdk​)。

下面我通过几何直观来拆解这些公式：

1. 基础预备：切线方程

在点 xk​ 处，如果我们沿着方向 dk​ 走，函数下降的最快初始斜率是：

斜率 g=∇f(xk​)Tdk​

（注：因为 dk​ 通常是下降方向，所以这个斜率 g 是一个负数。）

如果我们沿着这个斜率画一条直线（即 xk​ 点处的切线），它的方程是：

L(α)=f(xk​)+α⋅g

2. Armijo 准则（防止步长过大）

公式： f(xk​+αdk​)≤f(xk​)+c1​α⋅g

  • 直观解释： 右边其实是一条比切线稍微“平缓”一点的直线（因为 0<c1​<1）。

  • 为什么要这么写？ 真正的函数 f 可能是弯曲的。如果我们步子跨得太大，可能会跨过山谷跑到对面坡上去，导致函数值反而上升。

  • 你可以理解为： “我要求每走一步，函数值的下降量至少要达到‘理想下降量（切线方向）’的 c1​ 倍。”

3. Wolfe 准则（防止步长过小）

Armijo 条件有一个漏洞：如果我们每次只走“0.0000001”步，虽然函数确实下降了，但可能要走一万年才能到终点。所以需要 Wolfe 准则的第二条——曲率条件。

公式： ∇f(xk​+αdk​)Tdk​≥c2​⋅g

  • 直观解释： 左边是“新位置 xk+1​ 处的斜率”，右边是“起始位置 xk​ 处的斜率”乘以一个系数 c2​（比如 0.9）。

  • 为什么要这么写？ 起始斜率 g 是很大的负数。如果新位置的斜率仍然很大、很陡（比如还是 g 那么大），说明函数还在猛烈下降，你应该再多走几步。

  • 你可以理解为： “我要求到了新地方后，坡度必须变平缓了一些。如果还是很陡，说明我步子迈小了，还得继续往前走。”

4. Goldstein 准则（给步长套个“夹子”）

公式： f(xk​)+(1−c)αg≤f(xk+1​)≤f(xk​)+cαg

这个公式看起来最吓人，但其实最简单。它把新函数值 f(xk+1​) 限制在两条直线之间：

  1. 上限（右边）： 就是 Armijo 条件，防止步长太大。

  2. 下限（左边）： 另一条更陡的直线，防止步长太小。

  • 比喻： 就像买衣服砍价，Armijo 规定你出价不能比卖家标价高（防止太亏），而下限规定你出价不能比成本价低（防止卖家不卖，即算法不动）。

总结一下这些字母代表什么：

  • f(xk​): 当前山坡的高度。

  • f(xk+1​): 走完一步后的高度。

  • α: 步子跨多大（我们要算的）。

  • ∇f(xk​)Tdk​: 这一步出发时的“下坡斜率”。

  • c1​,c2​,c: 控制严格程度的“阀门”常数。

学习建议： 如果你是初学者，只需要记住回溯线搜索（Backtracking）： 先试一个大步长（α=1），如果不满足 Armijo 公式（下降不够），就给步长打个折（α=0.5α），直到满足为止。这种方法在工程上最常用，也最符合人类直觉。

我们将按照以下四个步骤来系统拆解：

  1. 降维打击： 把复杂问题变成一维几何题。

  2. 第一关卡（Armijo）： 也就是“上限控制”。

  3. 第二关卡（Curvature）： 也就是“下限控制”。

  4. 执行策略（Backtracking）： 实际怎么操作。

第一步：降维打击（把问题简化）

不管你的目标函数 f(x) 是多少维的（比如 1000 维的波浪），一旦确定了当前点 xk​ 和搜索方向 dk​，其实你就只在一条线上走了。

我们可以定义一个关于步长 α 的一维函数：

ϕ(α)=f(xk​+αdk​)

  • α=0 时，你在起点。

  • α>0 时，你沿着方向 dk​ 往前走。

  • ϕ′(0)=∇f(xk​)Tdk​：这是起点的斜率。记作 m。

    • 关键点： 因为我们要下山，所以 m 必定是负数（m<0）。

现在我们的任务变成了：在 α>0 的轴上，找一个点 α∗，让 ϕ(α∗) 足够小。

第二步：第一关卡——Armijo 准则（上限控制）

直观目的： 防止步子迈太大，跨过了山谷，甚至跳到了比起点还高的地方。我们需要函数值确实下降了，而且要降得对得起你的步长。

1. 公式拆解

f(xnew​)≤f(xold​)+c1​⋅α⋅m

也就是：

ϕ(α)≤ϕ(0)+c1​⋅α⋅ϕ′(0)

  • 左边 ϕ(α)：实际走了一步后的高度。

  • 右边 ϕ(0)+c1​αϕ′(0)：这是一条直线的方程。

    • 如果不乘 c1​，它就是起点的切线。

    • 因为 c1​ 是一个很小的正数（如 10−4），这条线的斜率比切线稍微平一点点，但依然是向下倾斜的。

2. 几何意义

请想象在起点画一条射线。

  • 切线是非常陡峭的下坡。

  • Armijo 线比切线稍微抬高了一点点。

  • 准则要求： 你的新高度必须在这条 Armijo 线下方。

3. 为什么要有 c1​？

如果 c1​=0，右边就是 f(xold​)，意味着只要函数值不增加就行。但这不够，我们需要它显著下降。 如果 c1​=1，右边就是切线。对于凸函数，曲线通常位于切线上方，那样可能永远找不到满足条件的点。 所以 c1​ 取极小值，既保证了下降，又给曲线留出了容错空间。

第三步：第二关卡——曲率/Wolfe 准则（下限控制）

直观目的： Armijo 只能防止步长太大，但如果你每次只挪动 0.00001 毫米，Armijo 也是满足的（因为确实下降了），但效率太低。我们需要防止步长太小。

1. 公式拆解

∇f(xnew​)Tdk​≥c2​⋅∇f(xold​)Tdk​

也就是：

ϕ′(α)≥c2​⋅ϕ′(0)

  • 左边 ϕ′(α)：走到新位置时的坡度（斜率）。

  • 右边 c2​⋅ϕ′(0)：初始坡度打个折。

    • 比如初始坡度是 -10（很陡）。

    • c2​=0.9。

    • 右边变成了 -9。

2. 几何意义（关键！）

这是一个关于斜率的比较，而不是高度。

  • 我们在起点时，坡度很陡（比如 -10）。

  • 如果我们走得不够远（步长太小），坡度依然很陡（比如还是 -9.9）。

  • 不等式要求：−9.9≥−9 是假的。说明还没达标，继续走！

  • 当我们走到谷底附近，坡度变平了（比如变成 -1，甚至变成正数 +2）。

  • 不等式要求：−1≥−9 是真的。达标了！

总结： Wolfe 准则要求新位置的坡度不能像起点那么陡。如果还是很陡，说明前面还有很大的下降空间，你得继续迈步。

第四步：强 Wolfe 准则（更严格的约束）

普通的 Wolfe 准则允许斜率变成很大的正数（比如冲过头到了对面的陡坡上）。为了更稳健，我们加上绝对值：

∣ϕ′(α)∣≤c2​∣ϕ′(0)∣

这意味着：

  1. 坡度不能太负（不能还在陡峭的下坡段）。

  2. 坡度也不能太正（不能冲到对面太陡的上坡段）。

  3. 结论： 也就是必须停在谷底平缓的区域。

系统总结图表

为了方便记忆，我们可以把这些准则看作是给 α 划定范围：

准则数学本质防止的问题物理直观
Armijo控制函数值的高度防止 α 太大 (Overshoot)“别跳到对面山上去了，必须真的往下降。”
Wolfe控制新位置的斜率防止 α 太小 (Undershoot)“如果脚下还是很陡，说明还没到底，继续往前走。”
Goldstein两条直线夹逼同时防止太大和太小“只能落在这一扇夹角区域里。”

实际操作：回溯线搜索 (Backtracking)

在工程实践（如写代码）中，同时满足 Armijo 和 Wolfe 计算比较麻烦（因为 Wolfe 需要算新点的导数，算导数很贵）。

最常用的妥协方案是：只保证 Armijo，通过策略间接避免步长太小。

算法流程（系统化描述）：

  1. 大胆假设： 先猜一个大步长，通常设 α=1（因为在牛顿法中 1 是理想值）。

  2. 小心求证： 检查 α=1 是否满足 Armijo 条件（高度是否降得够多？）。

    • Yes: 很好，接受这个步长。因为我们是从大往小试的，所以它通常不会太小。

    • No: 步子大了，扯着蛋了。

  3. 策略回撤： 把步长缩小一点，α←ρ⋅α （比如 ρ=0.5，即减半）。

  4. 循环： 回到第 2 步继续检查，直到满足为止。

为什么这个策略有效？ 因为它总是从“过大”开始缩减，一旦刚刚好满足 Armijo 条件就停止。这样找到的 α 往往是满足条件的最大值附近，因此自然避免了步长过小的问题，就不需要显式地去计算 Wolfe 条件了。

代码结构说明

  2. 目标函数：使用了一个简单的二次函数 f(x,y)=x2+10y2，这是一个典型的“狭长山谷”形状，如果不控制步长，很容易震荡。

C++ 代码实现

C++
#include <iostream>
#include <vector>
#include <cmath>
#include <functional>
#include <iomanip>

// ==========================================
// 1. 基础向量数学库 (为了代码可读性)
// ==========================================
using Vector = std::vector<double>;

// 向量加法: a + b
Vector operator+(const Vector& a, const Vector& b) {
    Vector res(a.size());
    for (size_t i = 0; i < a.size(); ++i) res[i] = a[i] + b[i];
    return res;
}

// 向量数乘: s * a
Vector operator*(double s, const Vector& a) {
    Vector res(a.size());
    for (size_t i = 0; i < a.size(); ++i) res[i] = s * a[i];
    return res;
}

// 向量点积: a . b
double dot(const Vector& a, const Vector& b) {
    double sum = 0.0;
    for (size_t i = 0; i < a.size(); ++i) sum += a[i] * b[i];
    return sum;
}

// 计算向量模长 (用于判断收敛)
double norm(const Vector& a) {
    return std::sqrt(dot(a, a));
}

// ==========================================
// 2. 目标函数定义
// ==========================================
// 我们使用函数 f(x, y) = x^2 + 10y^2
// 这是一个椭圆抛物面，极小值在 (0, 0)

double func(const Vector& x) {
    return x[0] * x[0] + 10.0 * x[1] * x[1];
}

Vector gradient(const Vector& x) {
    // df/dx = 2x, df/dy = 20y
    return { 2.0 * x[0], 20.0 * x[1] };
}

// ==========================================
// 3. 核心：回溯线搜索 (Backtracking Line Search)
// ==========================================
// 这里的逻辑严格遵循 Armijo 准则
double backtracking_line_search(
    const Vector& x_curr,       // 当前位置
    const Vector& d,            // 搜索方向 (通常是负梯度)
    const Vector& grad,         // 当前点的梯度
    std::function<double(const Vector&)> f // 目标函数句柄
) {
    // --- 超参数设置 ---
    double alpha = 1.0;     // 初始步长 (牛顿法通常设为1，梯度下降也可以从1开始)
    double rho = 0.5;       // 衰减因子 (步长缩小的倍率)
    double c = 1e-4;        // Armijo 常数 (只要有一点点下降就行)

    // 计算当前函数值
    double f_curr = f(x_curr);
    
    // 计算初始斜率 (Gradient . Direction)
    // 注意：因为 d 是负梯度方向，所以 slope 应该是一个负数
    double slope = dot(grad, d);

    // --- Armijo 循环 ---
    // 公式: f(x + alpha*d) > f(x) + c * alpha * slope
    // 如果“新高度” 大于 “目标下降高度”，说明步长太大了，需要缩小
    while (f(x_curr + alpha * d) > f_curr + c * alpha * slope) {
        
        // 记录一下调试信息（可选）
        // std::cout << "  [Search] alpha=" << alpha << " 不满足 Armijo, 缩小..." << std::endl;
        
        alpha *= rho; // 步长减半
        
        // 防止死循环（浮点数精度保护）
        if (alpha < 1e-8) {
            std::cout << "  [Warning] 步长过小，强制退出线搜索" << std::endl;
            break; 
        }
    }

    return alpha;
}

// ==========================================
// 4. 主优化循环 (梯度下降法)
// ==========================================
int main() {
    // 初始猜测点 (10, 10)
    Vector x = { 10.0, 10.0 };
    int max_iter = 100;
    double tol = 1e-6; // 收敛容差

    std::cout << std::fixed << std::setprecision(5);
    std::cout << "开始优化: f(x,y) = x^2 + 10y^2" << std::endl;
    std::cout << "初始位置: (" << x[0] << ", " << x[1] << ")" << std::endl;
    std::cout << "-------------------------------------------------------------" << std::endl;

    for (int k = 0; k < max_iter; ++k) {
        Vector g = gradient(x);
        
        // 检查收敛
        if (norm(g) < tol) {
            std::cout << "梯度接近 0，收敛成功！" << std::endl;
            break;
        }

        // 1. 确定搜索方向 (梯度下降法使用负梯度)
        Vector d = -1.0 * g; 

        // 2. 调用非精确线搜索寻找 alpha
        //    这就是把“一维搜索”剥离出来的体现
        double alpha = backtracking_line_search(x, d, g, func);

        // 3. 更新位置
        x = x + alpha * d;

        // 打印每一步的信息
        std::cout << "Iter " << k+1 
                  << " | f(x) = " << func(x) 
                  << " | Step(alpha) = " << alpha 
                  << " | Pos = (" << x[0] << ", " << x[1] << ")" << std::endl;
    }

    std::cout << "-------------------------------------------------------------" << std::endl;
    std::cout << "最优解: (" << x[0] << ", " << x[1] << ")" << std::endl;

    return 0;
}

代码关键点深度解析

1. 那个 while 循环 (代码 77行)

C++
while (f(x_curr + alpha * d) > f_curr + c * alpha * slope)

这行代码就是 Armijo 公式 的直接翻译：

  • 左边：f(x_curr + alpha * d) 是试探点的新函数值 f(xk​+αdk​)。

  • 右边：f_curr + c * alpha * slope 是直线方程 f(xk​)+c⋅α⋅∇fTdk​。

  • 逻辑：只要“新值”还比“直线值”大（即位于直线上方），就说明我们没有降得足够多（或者步子太大跨到对面去了），所以要执行 alpha *= rho。

2. 为什么需要 dot(grad, d)? (代码 72行)

这里 slope 也就是 g=∇f(xk​)Tdk​。

  • 在梯度下降中， d=−∇f，所以 slope 实际上是 −∣∣∇f∣∣2。

  • 这保证了 slope 永远是负数。

  • Armijo 条件右边的 c * alpha * slope 因此也是负数，代表要求函数值必须减小一定的量。

3. 初始 alpha 的选择

在代码中我设置了 double alpha = 1.0;。

  • 这是标准的做法。回溯搜索策略是**“从大往小试”**。

  • 如果一开始就试一个小步长（比如 0.001），虽然肯定满足条件，但收敛极慢。

  • 从 1.0 开始试，如果运气好（比如在平坦区域），就能走一大步；如果运气不好（在陡峭区域），循环会自动把它缩小到 0.5, 0.25... 直到合适为止。

观察输出结果

Plaintext
Iter 1 | f(x) = ... | Step(alpha) = 0.06250 | Pos = ...

你会发现 Step(alpha) 是动态变化的。有时候它是 0.125，有时候可能是 0.0625。这正是非精确搜索的魅力——它根据地形自动调整步伐，比固定学习率（Fixed Learning Rate）要聪明得多。

你可以尝试修改 func 函数中的 10.0 为 100.0（让山谷更窄），你会发现 alpha 会自动变得更小以适应更陡峭的地形。
