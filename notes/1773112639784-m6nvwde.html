  1. ROS 可配置参数：你在 launch / yaml 里能设的

  2. 类成员变量：节点内部长期保存的状态

这份代码的核心目标是：估计 camera_frame 相对于 base_frame 的旋转外参 roll/pitch/yaw，平移不变。整体说明、辅助函数、结构体、构造参数和优化入口都在文件前半段和 runCalibration() 里。

1. ROS 可配置参数

这些参数是在构造函数里通过 pnh_.param(...) 读进来的。也就是你最可能在 launch 或 yaml 里改的那一批。

1.1 话题和坐标系参数

cloud_topic_

对应参数名：

cloud_topic

默认值：

"/left_rgbd/depth/points"

含义：

  • 订阅深度相机点云的话题。

作用：

  • cloudCallback() 从这个话题收点云。

scan_topic_

对应参数名：

scan_topic

默认值：

"/2dlidar_scan_filtered"

含义：

  • 订阅 2D 雷达扫描数据的话题。

作用：

  • scanCallback() 从这里拿到激光扫描。

base_frame_

对应参数名：

base_frame

默认值：

"base_link"

含义：

  • 目标参考坐标系。

  • 所有几何约束最终都在这个坐标系下成立。

作用：

  • 点云先被变换到 base_frame_

  • scan 也被变换到 base_frame_

  • 地面法向量目标是 [0,0,1]，也是在这个坐标系下定义的。

camera_frame_

对应参数名：

camera_frame

默认值：

"left_rgbd_link"

含义：

  • 被标定的 RGBD 相机主坐标系。

作用：

  • 初始 TF 是通过 lookupTransform(base_frame_, camera_frame_, ...) 读出来的。

  • 优化变量 rpy 代表的就是这个 camera_frame -> base_frame 的旋转。

1.2 采样与滤波参数

num_frames_

对应参数名：

num_frames

默认值：

30

含义：

  • 需要累计多少帧点云和多少帧激光，才开始优化。

作用：

  • cloud_count_ >= num_frames_

  • scan_count_ >= num_frames_
    两者都满足时才开始标定。

经验上：

  • 大一点更稳

  • 太大则采集时间更长，也可能把场景变化引入进来

voxel_leaf_

对应参数名：

voxel_leaf

默认值：

0.02

单位：

  • 米

含义：

  • 点云体素降采样的体素边长。

作用：

  • VoxelGrid 下采样时 x/y/z 三个方向都用这个值。

直观理解：

  • 越大，点更稀疏，计算更快，但平面精度可能下降

  • 越小，细节更多，但噪声和计算量也更大

z_min_, z_max_

对应参数名：

z_minz_max

默认值：

0.15.0

单位：

  • 米

注意：
虽然变量名叫 z_min/z_max，但这里实际不是按 z 轴裁剪，而是按“点到传感器原点的欧氏距离”做筛选。代码里是算 dist = sqrt(dx²+dy²+dz²)。

含义：

  • 只保留离传感器距离在 [z_min_, z_max_] 范围内的点。

作用：

  • 去掉太近的噪声点和太远的不可靠点。

这块变量名略有误导，严格说更像：

  • range_min

  • range_max

1.3 平面 / 直线拟合参数

ransac_dist_thresh_

对应参数名：

ransac_dist_thresh

默认值：

0.02

单位：

  • 米

含义：

  • 平面 RANSAC 的点到平面距离阈值。

作用：

  • 在 fitConstrainedPlane() 里传给：

seg.setDistanceThreshold(ransac_dist_thresh_);

用来决定一个点算不算平面内点。

直观理解：

  • 小：更严格，抗噪差时容易失败

  • 大：更宽松，容易把非平面点也吃进去

perp_dot_thresh_

对应参数名：

perp_dot_thresh

默认值：

0.15

含义：

  • 地面法向量和墙面法向量的垂直性容差，使用点积的绝对值判断。

代码逻辑：

double dot = std::abs(wall.normal.dot(ground.normal));if (dot > perp_dot_thresh_) skip;

直观理解：

  • 理想垂直时点积应接近 0

  • 0.15 大约对应夹角离 90° 还有一点容忍

作用：

  • 防止“找到的墙”其实不是垂直墙

  • 属于数据质量门槛

line_ransac_dist_

对应参数名：

line_ransac_dist

默认值：

0.03

单位：

  • 米

含义：

  • 2D 直线 RANSAC 的内点距离阈值。

作用：

  • fitLine2D() 里点到候选直线的距离若小于这个阈值，则算内点。

直观理解：

  • 越小越严格

  • 越大越容易吃进杂点

min_line_inliers_

对应参数名：

min_line_inliers

默认值：

10

含义：

  • 2D 激光墙线拟合后，至少要有多少内点才接受。

作用：

  • 如果激光拟合出的墙线支持点太少，就丢弃这一帧 scan。

1.4 输入检查参数

expected_cloud_frame_

对应参数名：

expected_cloud_frame

默认值：

""

含义：

  • 期望点云消息的 frame_id。

  • 空字符串表示不检查。

作用：

  • 如果实际 msg->header.frame_id 不等于这个值，就跳过该帧。

用途：

  • 防止你订到了错误的话题，或者点云 frame 在运行时变掉

expected_scan_frame_

对应参数名：

expected_scan_frame

默认值：

""

含义和作用同上，只不过针对 laser scan。

1.5 优化权重参数

weight_ground_

对应参数名：

weight_ground

默认值：

10.0

含义：

  • 地面法向量残差的权重。

作用：

  • 在 Ceres 里通过 ScaledLoss 加权。

影响：

  • 越大，优化越优先满足“地面朝上”

  • 主要影响 roll / pitch

weight_wall_az_

对应参数名：

weight_wall_az

默认值：

5.0

含义：

  • 墙面方位角与激光方位角匹配项的权重。

作用：

  • 控制 yaw 约束的强弱。

weight_wall_vert_

对应参数名：

weight_wall_vert

默认值：

2.0

含义：

  • 墙面竖直性约束的权重。

作用：

  • 辅助限制墙法向量旋转后 z 分量接近 0。

2. 类成员变量

这些不是 launch 参数，而是节点内部一直维护的状态。定义在 private: 区域。

2.1 ROS 相关成员

nh_, pnh_

  • nh_：普通命名空间句柄

  • pnh_：私有命名空间句柄

作用：

  • nh_ 用来订阅/发布

  • pnh_ 用来读私有参数

tf_buffer_, tf_listener_

作用：

  • 从 TF 树查询变换。

  • 比如：

    • cloud frame -> base

    • scan frame -> base

    • camera frame -> base

cloud_sub_, scan_sub_

作用：

  • 订阅点云和激光话题。

2.2 数据采集状态

mtx_

作用：

  • 互斥锁，保护回调中的共享数据。

cloud_count_, scan_count_

含义：

  • 已经收集到多少帧有效点云 / 有效 scan。

calibration_done_

含义：

  • 标定是否已经完成。

作用：

  • 完成后停止继续收集新数据。

2.3 累计观测量

ground_planes_

类型：

std::vector<PlaneParam>

含义：

  • 每帧拟合到的地面参数集合。

  • 注意其中法向量已经被转成了 camera frame。

wall_planes_

含义：

  • 每帧拟合到的墙面参数集合。

  • 同样保存的是 camera frame 下的法向量。

lidar_azimuths_

类型：

std::vector<double>

含义：

  • 每帧激光拟合出的墙法向方位角。

  • 这个角是在 base_link 的 x-y 平面 中定义的。

2.4 用于跨帧消除符号二义性的参考量

have_first_wall_normal_

含义：

  • 是否已经记录第一帧墙法向量。

first_wall_normal_cam_

含义：

  • 第一帧墙法向量，位于 camera frame。

  • 后续帧拿它来统一墙法向量的方向。

have_first_lidar_azimuth_

first_lidar_azimuth_

含义：

  • 第一帧 lidar 墙法向角及其有效标志。

  • 用来避免后续帧的法向角突然跳 π。

2.5 初始 TF 相关成员

got_initial_tf_

含义：

  • 是否成功读到初始 TF。

initial_translation_

含义：

  • 初始 TF 的平移 (tx, ty, tz)。

注意：

  • 这份代码不优化平移

  • 最终结果里平移直接沿用它。

initial_quaternion_

含义：

  • 初始 TF 的四元数旋转。

作用：

  • 用来得到 initial_rpy_

  • 用来把平面法向量从 base 转回 camera

  • 也是最终比较“优化前 / 优化后”时的原始姿态参考。

initial_rpy_

含义：

  • 从初始 TF 提取的欧拉角 (roll, pitch, yaw)。

作用：

  • 作为 Ceres 的初值。

initial_tf_msg_

含义：

  • 原始的 geometry_msgs::TransformStamped 形式的初始 TF。

2.6 可视化相关成员

visualizing_

含义：

  • 是否已进入标定后的 RViz 可视化模式。

T_calibrated_

含义：

  • 标定后的变换，注释写的是 base_link -> camera_frame。

不过要注意：
从构造方式看，它存的是“平移 + 优化后旋转”的仿射变换，并被直接拿来做 T_total * pt。所以你在理解时要始终以“代码实际乘法方向”为准。

T_original_

含义：

  • 原始 TF 对应的仿射变换。

cloud_native_frame_

含义：

  • 记住原始点云消息最初的 frame_id，方便可视化链路使用。

viz_cloud_sub_, viz_scan_sub_

  • 标定后重新订阅用于显示前后效果。

pub_cloud_before_, pub_cloud_after_, pub_lidar_cloud_

含义：

  • 发布“优化前点云”“优化后点云”“激光参考点云”。

static_tf_broadcaster_

作用：

  • 发布 camera_frame + "_calibrated" 的静态 TF。

debug_log_

含义：

  • 保存完整调试日志，最后一起输出。

3. 数据结构里的字段

3.1 PlaneParam

定义：

struct PlaneParam {    Eigen::Vector3d normal;    double d;    int inlier_count;};

normal

含义：

  • 平面法向量 (nx, ny, nz)，通常已归一化。

平面方程：

nxx+nyy+nzz+d=0n_x x + n_y y + n_z z + d = 0nx​x+ny​y+nz​z+d=0

d

含义：

  • 平面方程中的常数项。

注意：

  • 这个量会随着法向量翻转一起翻转：

    • normal -> -normal

    • d -> -d
      代码里确实这样做了。

inlier_count

含义：

  • 拟合这个平面的内点数量。

作用：

  • 后面平均法向量时拿它做权重。

3.2 LineParam

定义：

struct LineParam {    Eigen::Vector2d direction;    double normal_azimuth;    int inlier_count;};

direction

含义：

  • 2D 直线方向向量，单位向量。

normal_azimuth

含义：

  • 直线法向量在 x-y 平面内的方位角：

atan2(ny,nx)atan2(n_y, n_x)atan2(ny​,nx​)

作用：

  • 这是 wall azimuth cost 的目标量。

inlier_count

含义：

  • 拟合出这条线时的内点数量。

作用：

  • 作为 scan 质量门槛。

4. 优化相关参数

这一部分最关键。

4.1 rpy

在 runCalibration() 里：

double rpy[3] = { initial_rpy_.x(), initial_rpy_.y(), initial_rpy_.z() };

含义：

  • 优化变量，长度 3：

    • rpy[0] = roll

    • rpy[1] = pitch

    • rpy[2] = yaw

它表示的是：

  • 最终待估计的 camera_frame -> base_frame 旋转

不是增量，而是最终旋转本身。代码注释也直接这么写了。

4.2 ground_normal_cam

Eigen::Vector3d ground_normal_cam = averageNormals(ground_planes_, "ground");

含义：

  • 多帧平均后的地面法向量，位于 camera frame。

作用：

  • GroundNormalCost 的输入。

4.3 wall_normal_cam

含义：

  • 多帧平均后的墙法向量，位于 camera frame。

作用：

  • WallAzimuthCost 和 WallVerticalityCost 的输入。

4.4 lidar_wall_azimuth

含义：

  • 多帧平均后的激光墙法向方位角，位于 base_link 的 x-y 平面。

作用：

  • WallAzimuthCost 的目标值。

4.5 roll_opt, pitch_opt, yaw_opt

含义：

  • 优化后的最终欧拉角结果。

它们会再转回旋转矩阵 R_opt，以及四元数 q_opt。

4.6 R_opt

含义：

  • 由最终 roll_opt/pitch_opt/yaw_opt 组成的旋转矩阵。

作用：

  • 用来验证：

    • 地面是否被转正到 z 轴

    • 墙面是否水平法向正确

    • 墙方位角是否与 lidar 一致

5. 三个 cost 里的参数

5.1 GroundNormalCost

定义里只有一个成员：

n_ground_cam

含义：

  • 地面法向量，位于 camera frame。

operator() 里的局部量：

cr, sr

  • cos(roll), sin(roll)

cp, sp

  • cos(pitch), sin(pitch)

cy, sy

  • cos(yaw), sin(yaw)

R00 ~ R22

  • 手动展开的旋转矩阵元素，对应

R=Rz(yaw)Ry(pitch)Rx(roll)R = R_z(yaw)R_y(pitch)R_x(roll)R=Rz​(yaw)Ry​(pitch)Rx​(roll)

nx, ny, nz

  • 输入法向量的三个分量

rx, ry, rz

  • 旋转后的法向量分量，即

R⋅nground_camR \cdot n_{ground\_cam}R⋅nground_cam​

residual[0], residual[1], residual[2]

  • 残差，目标是让旋转后的法向量变成 [0,0,1]。

5.2 WallAzimuthCost

n_wall_cam

  • 相机系下墙法向量。

lidar_azimuth

  • 激光测得的墙法向角，单位 rad。

局部量里：

cam_az

  • 旋转后墙法向量在 base_link x-y 平面的方位角：

atan2(ry,rx)atan2(ry, rx)atan2(ry,rx)

diff

  • 相机推导出的墙方位角，与激光方位角的差

residual[0] = sin(diff)

  • 用 sin 而不是直接 diff，是为了处理角度绕回问题。

5.3 WallVerticalityCost

n_wall_cam

  • 相机系下墙法向量。

局部量里：

R20, R21, R22

  • 旋转矩阵的第三行

rz

  • 旋转后墙法向量的 z 分量

residual[0] = rz

  • 希望墙法向量 z 分量接近 0，也就是墙是竖直平面。

6. 容易混淆的几个参数

这里我专门指出几个最容易误解的。

z_min_ / z_max_

名字像按 z 轴裁剪，实际上是按点到传感器的距离筛选。

initial_quaternion_ / initial_rpy_

它们不是优化结果，而是：

  • 从现有 TF 树中读出的初始外参旋转

  • 给优化提供初值和法向量回转参考。

weight_ground_ / weight_wall_az_ / weight_wall_vert_

它们不是直接乘残差，而是通过 ScaledLoss 加进去，控制三类约束的相对重要性。

ground_planes_ / wall_planes_

虽然平面是一开始在 base_link 下拟合出来的，但真正存进去用于优化的法向量，已经被转回 camera frame 了。

    • ground_normal_cam, wall_normal_cam, lidar_wall_azimuth
